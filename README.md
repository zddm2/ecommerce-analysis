# 全球跨境电商经营数据分析与业务诊断

## 项目简介
本项目基于 Kaggle 全球电商交易数据集（100万+订单，62个字段，506MB），使用 Python（pandas、numpy）和 SQL 完成从数据清洗、指标体系搭建到业务诊断的全流程分析。最终输出可落地的运营建议，并将核心分析逻辑用 SQL 复现验证。

**技术栈**：Python (pandas, numpy) | SQL (SQLite) | Git

## 数据集说明
- 来源：Kaggle – 全球电子商务数据集（2024-2026）
- 记录数：1,000,000+ 条（清洗后保留 200,000 行样本）
- 主要字段：
  - 订单信息：`order_id`, `order_date`, `total_price_usd`
  - 用户信息：`customer_id`, `customer_segment`, `age`
  - 产品信息：`category`, `brand`, `unit_price_usd`, `quantity`
  - 物流信息：`delivery_days`, `shipping_cost_usd`
  - 评价信息：`rating`, `return_reason`

> 注：原始数据文件（~506MB）未上传至 GitHub，清洗后的样本文件 `ecommerce_clean_sample.csv` 可联系作者获取或自行运行清洗脚本生成。



## 分析流程
1. **数据清洗**  
   - 分块读取（chunk size=10,000）处理 506MB 数据  
   - 缺失值处理：关键字段删行，数值列中位数填充，分类列填 'Unknown'  
   - 异常值过滤：单价 10-500 美元，数量 1-1000 件，利润≥0，年龄 18-75 岁  
   - 清洗后保留 20 万高质量样本

2. **指标体系构建**  
   - 整体大盘：GMV（$80.66M）、客单价（$404）、复购率（0.16%）  
   - 用户分层：Regular (60.2%)、Premium (30.0%)、VIP (9.8%)  
   - 高价值用户：Top 10% 贡献 31.3% 销售额

3. **深度诊断**  
   - 物流分析：平均配送 7.5 天，50% 订单 >7 天  
   - 退货分析：退货率 10.09%，主因“其他地方价格更优”  
   - 折扣分析：67% 销售额来自折扣订单，但未带来复购  
   - RFM 模型：识别出 2.2 万高价值流失用户（潜在价值 $2.24M）

4. **SQL 验证**  
   使用 SQLite 编写 8 条查询（含窗口函数、CTE），复现核心指标，结果与 Python 完全一致。

## 核心发现与建议
### 发现
- 复购率仅 0.16%，几乎全部用户只购买一次  
- 物流体验差：半数订单配送超过 7 天  
- 退货主因：其他平台价格更优  
- 2.2 万高价值用户已流失，贡献 $2.24M 潜在召回价值

### 建议
- 物流优化：将平均配送天数压缩至 5 天内，超时订单自动补偿优惠券  
- 价格策略：建立价格匹配机制，减少因价格导致的退货  
- 用户召回：向流失高价值用户推送专属大额券（满 500 减 100）  
- 复购激励：首购后 7 天内推送复购 8 折券，目标提升复购率至 5% 以上
